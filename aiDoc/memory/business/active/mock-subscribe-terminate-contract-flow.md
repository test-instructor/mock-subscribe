# 用户业务记忆

- 需求：为 MockSubscribe 补齐“申请解约”接口链路，保持与签约配置模式一致，并支持 Python 脚本发起解约。
- 细节：商户配置需要新增独立的解约目标状态、解约状态延时、解约回调开关、解约回调延时字段，不直接复用 `sign*` 字段。
- 细节：解约后需要按配置等待，再更新用户协议状态，并按签约同源的 `notify_url` 发送协议变更回调。
- 细节：解约流程需要写入 `terminate` 操作记录，保存响应 XML、回调结果和回调时间，保持与签约/扣款链路一致。
- 细节：Python 脚本需要新增独立的解约请求构造与命令入口，调用 `/papay/deletecontract` 并使用签约相同回调地址配置。
